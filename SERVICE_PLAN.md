# 정리의 숲 서비스 기획서

## 1. 서비스 개요

- **목적:** 흩어진 메모를 연결하고 구조화하는 경험을 통해, 옵시디언 초보자가 지식 관리 습관을 시작하도록 돕는다.
- **타겟 사용자:** 메모는 많이 하지만 분류·연결 방법이 낯선 대학생, 직장인, 옵시디언 입문자.
- **핵심 가치:** 사용자가 적은 내용에서 AI가 연결·태그·정리 방식을 제안해 다음 행동을 쉽게 만든다.

## 2. 페이지 구성

| 페이지 | 역할 | 주요 행동 |
| --- | --- | --- |
| 홈 (`index.html`) | 서비스 소개와 사용 기록 확인 | 각 기능 페이지로 이동, 테마 전환 |
| 아이디어 그래프 (`workspace.html`) | 노트 간 의미 연결 탐색 | 노트 추가, AI 연결 분석, 마크다운 복사 |
| AI 도구 (`tools.html`) | AI 기반 노트 정리·질문 응답 | 메모 정리, 옵시디언 질문 |
| 가이드·FAQ (`guide.html`) | 개념 학습과 불안 해소 | Vault·태그·내부링크 학습, FAQ 확인 |

## 3. AI 기능 설계

| 기능 | 입력 | 출력 | 실패 처리 |
| --- | --- | --- | --- |
| 연결 분석 | 2~20개 노트의 제목·내용 | 연결된 노트 쌍, 연결 이유, 추천 태그, 그래프 | 노트 수 부족·잘못된 요청·API 오류·30초 지연 안내 |
| 노트 정리기 | 최대 5,000자 메모 | frontmatter, 태그, 제목, 본문, 내부링크 후보가 있는 마크다운 | 빈 입력·API 오류·30초 지연 안내 |
| Q&A 챗봇 | 최대 500자 옵시디언 질문과 최근 대화 | 초보자 눈높이의 한국어 답변 | 빈 입력·API 오류·30초 지연 안내 |

## 4. 보너스 기능

- 사용자가 선택한 다크/라이트 테마를 브라우저 localStorage에 저장한다.
- 방문 수, 저장 노트 수, AI 성공 호출 수를 같은 브라우저에서만 집계한다. 외부 분석 서비스나 서버로 전송하지 않는다.
- hover, 로딩, 복사 완료 피드백으로 사용자의 행동 결과를 바로 보여준다.
